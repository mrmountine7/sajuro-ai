# 사주로(sajuro.ai)  
## 피그마 Make 업로드용 통합 마스터 프롬프트

---

## 1. 목적

이 문서는 피그마 Make에 업로드하거나 입력하여, **사주로(sajuro.ai) 앱의 UI/UX 개선 방향**에 맞는 화면을 보다 일관되게 생성하기 위한 **통합 마스터 프롬프트**이다.

이 프롬프트의 목표는 다음과 같다.

- 사주로 앱의 전체 정보구조와 화면 철학을 반영
- 카드형 분석 허브, 명식 보관소, 분석 결과, 기록, 마이페이지 구조를 일관되게 생성
- 피그마에서 바로 활용 가능한 모바일 앱 화면 중심 결과물 유도
- 밝은 탐색형 화면 + 어두운 몰입형 화면의 혼합 전략 반영
- 한국어 기반 서비스 카피와 사주 서비스 특성을 유지

---

## 2. 사용 방법

이 문서는 다음 방식으로 활용한다.

### 권장 방식
1. 본 문서를 **피그마 Make의 상위 지시문**으로 입력한다.
2. 별도로 만들 화면이 있다면, 이 문서 아래에 **추가 화면 지시문**을 붙인다.
3. 한 번에 전체 앱을 만들려고 하기보다, 아래 우선순위로 나누어 생성한다.
   - 홈
   - 분석 허브
   - 보관소
   - 명식 상세
   - 분석 결과
   - 기록
   - 마이페이지

### 주의
- 이 문서는 **디자인 방향과 구조를 통제하는 마스터 프롬프트**이다.
- 정교한 결과를 원하면 화면별 프롬프트를 별도로 추가해야 한다.
- 기존 md 문서인 **컴포넌트 네이밍/props 표준안** 및 **Variables 토큰 표준안**과 함께 쓰면 가장 좋다.

## 3. 피그마 Make 통합 마스터 프롬프트

아래 내용을 그대로 사용한다.

---

너는 한국의 프리미엄 운세/사주 모바일 앱 UI 디자이너이자 프로덕트 디자이너다.  
다음 조건에 따라 **사주로(sajuro.ai)** 앱의 모바일 화면을 설계해줘.

### A. 서비스 개요

사주로는 한국어 기반의 AI 사주 분석 앱이다.  
단순한 무료 운세 앱이 아니라, **저장된 명식을 기반으로 내 사주, 테마별 운세, 궁합, 분석 결과 기록, 질문/답변 이력까지 관리하는 프리미엄형 서비스**다.

이 앱의 핵심 가치는 다음과 같다.

- 저장된 명식을 다시 불러와 빠르게 분석할 수 있다.
- 분석 메뉴가 카드형으로 정리되어 있어 탐색이 쉽다.
- 분석 결과를 읽고 끝나는 것이 아니라, 결과에 대해 추가 질문을 남기고 답변 기록을 축적할 수 있다.
- 마이페이지는 단순한 설정창이 아니라, 내 명식/분석기록/질문답변 내역을 모아두는 개인 아카이브다.

### B. 전체 UX 방향

앱은 **전면 라이트 모드 기반의 통일된 카드 UX**를 지향한다.

1. **전체 화면은 밝고 정돈된 카드 UX**
   - 예: 홈, 분석 허브, 보관소, 명식 상세, 분석 결과, 기록, 마이페이지
   - 배경은 밝은 회색 또는 아주 연한 중성색
   - 카드들이 보기 좋게 정렬된 구조
   - 서비스 메뉴가 한눈에 들어와야 함
   - 프리미엄 감성은 골드/로즈 포인트 컬러와 절제된 타이포로 표현

### C. 반드시 반영해야 하는 핵심 구조

앱의 하단 탭 구조는 아래 5개를 기준으로 한다.

- 홈
- 분석
- 보관소
- 기록
- 마이

### D. 핵심 화면 정의

#### 1) 홈
목적: 사용자가 앱에 들어오자마자 오늘 볼 만한 운세와 최근 본 분석을 빠르게 이어보게 한다.

반드시 포함:
- 인사 문구
- 오늘의 한 줄 흐름 카드
- 추천 분석 카드 4개 내외
- 최근 본 분석
- 저장된 명식 바로가기

#### 2) 분석
목적: 기존 숨겨진 메뉴 구조를 대체하는 **카드형 분석 허브**

반드시 포함:
- 상단 제목: 분석 메뉴
- 설명 문구: 보고 싶은 분석을 선택해보세요
- 카테고리 필터: 전체 / 내 사주 / 테마운 / 궁합·관계 / 생활·부가 / 이미지 분석
- 2열 카드형 메뉴
- 각 카드에는 아이콘, 제목, 설명 1줄, 배지(무료/추천/인기/준비중 등) 포함

분석 카드 예시:
- 만세력
- 내 사주 정밀분석
- 평생운세
- 운세 흐름
- 신년운세
- 토정비결
- 금전운
- 애정운
- 직업운
- 사업운
- 건강운
- 친구운
- 연인궁합
- 결혼궁합
- 재회 가능성
- 상대 속마음
- 가족 궁합
- 꿈해몽
- 이름풀이
- 작명소
- 택일
- MBTI
- 손금 보기
- 관상 보기

#### 3) 보관소
목적: 저장된 명식을 관리하고 빠르게 분석에 진입하게 한다.

반드시 포함:
- 제목: 명식 보관소
- 내 프로필 카드
- 즐겨찾기 영역
- 저장된 명식 리스트
- 각 명식 카드에는 이름, 생년월일시, 관계 태그, 빠른 버튼(분석 / 궁합 / 더보기)

#### 4) 명식 상세
목적: 저장된 명식의 핵심 정보를 보기 좋게 요약하고 운세 풀이 진입 CTA를 제공한다.

반드시 포함:
- 본인 정보 카드
- 오행 요약 카드
- 일주 동물/타고난 성향 같은 요약 카드
- 사주 격자(시주/일주/월주/년주)
- 하단 고정 CTA: 운세 풀이 보기

명식 상세 화면은 첨부 예시처럼 정보 밀도가 있더라도 정돈된 구조여야 하며, 너무 복잡하거나 난잡해 보이면 안 된다.

#### 5) 분석 결과
목적: 결과를 읽고, 저장하고, 추가 질문하도록 유도한다.

반드시 포함:
- 상단 결과 요약 카드
- 핵심 키워드 태그
- 총평 / 강점 / 주의점 / 시기 포인트 / 실행 조언 섹션
- 하단 액션: 추가 질문하기 / 핵심만 요약 / 쉽게 다시 설명 / 관련 분석 추천

#### 6) 기록
목적: 분석 결과와 질문/답변 이력을 다시 볼 수 있게 한다.

반드시 포함:
- 필터: 전체 / 내 분석 / 궁합 / 질문답변 / 저장한 결과
- 카드 리스트
- 각 카드에는 제목, 날짜, 한 줄 요약, 다시보기 버튼 포함

#### 7) 마이페이지
목적: 내 데이터 관리 허브

반드시 포함:
- 프로필 요약 카드
- 명식 보관소 / 즐겨찾기 / 최근 본 분석
- 분석 기록 / 저장 결과 / 질문답변 내역
- 구독 / 결제 / 알림 / 문의 메뉴

### E. 카드 UI 스타일

카드는 이 앱의 핵심이다.  
다음 기준을 꼭 지켜라.

- 모바일 기준 2열 카드 구조를 기본으로 한다.
- 카드 높이는 균형감 있게 맞춘다.
- 카드 내부 여백은 넉넉하게 준다.
- 제목은 선명하고 설명은 1줄 또는 2줄 이내로 짧게 유지한다.
- 카드에는 아이콘이 포함되어야 한다.
- 카드 배경은 너무 화려하지 않게 절제된 느낌을 유지한다.
- 카드 간 간격과 정렬이 매우 중요하다.
- 화면 전체가 “잘 정돈된 프리미엄 사주 앱”처럼 보여야 한다.

### F. 비주얼 스타일

다음 키워드를 시각적으로 반영해라.

- 프리미엄
- 정돈됨
- 따뜻한 운세 감성
- 카드형 탐색
- 한국어 서비스
- 너무 싸구려스럽지 않음
- 과한 전통문양 남발 금지
- 현대적이되 사주 서비스 분위기는 유지

#### 전면 라이트 모드
- 매우 옅은 회색 배경
- 흰색 카드
- 진한 텍스트
- 포인트 컬러는 골드/옐로우/블루/로즈 사용 가능
- 모든 화면이 동일한 밝은 톤으로 통일

### G. 컬러 방향

아래 분위기를 따른다.

- 앱 기본 라이트 배경: 아주 연한 회색
- 앱 기본 다크 배경: 깊은 블랙 브라운
- 카드 라이트 배경: 화이트 또는 매우 옅은 회색
- 카드 다크 배경: 브라운 블랙
- 포인트 컬러: 골드, 옐로우, 소프트 블루, 로즈베이지
- 상태 배지:
  - 무료: 보라 계열
  - 현재 무료: 민트/그린 계열
  - 프리미엄: 골드
  - 준비중: 회색

### H. 타이포 방향

- 한국어 가독성이 최우선이다.
- 제목과 본문 위계를 분명히 구분한다.
- 카드 제목은 선명하고 짧게
- 카드 설명은 1줄 또는 2줄로 제한
- 메타 정보는 너무 작지 않게 처리
- 숫자와 날짜도 깔끔하게 보이게 한다.

### I. 콘텐츠 카피 방향

카드 제목과 설명은 한국어로 작성한다.  
문구는 너무 길지 않아야 하며, 사용자에게 기능이 바로 이해되어야 한다.

예시 스타일:
- 내 사주 정밀분석  
  타고난 기질과 인생 구조를 깊게 해석
- 운세 흐름  
  대운·세운·월운의 변화 포인트 확인
- 금전운  
  재물 흐름과 지출 위험을 분석
- 연인궁합  
  끌림, 갈등, 감정 리듬을 확인

### J. 컴포넌트 구성 원칙

필요한 화면을 만들 때 아래 컴포넌트를 일관되게 사용해라.

- 상단 헤더
- 카드
- 배지
- 버튼
- 리스트
- 바텀시트
- 태그/칩
- 하단 탭바

각 화면은 별개가 아니라 같은 앱처럼 보여야 한다.  
즉, 모든 화면은 동일한 디자인 시스템으로 설계되어야 한다.

### K. 피그마 결과물 형태

다음 형태로 결과를 만들어라.

- 모바일 앱 프레임
- 한국어 텍스트 포함
- 실제 앱처럼 보이는 high-fidelity 수준
- 동일한 UI 시스템을 공유하는 여러 화면 세트
- 화면별 제목과 콘텐츠가 구체적으로 들어간 형태
- 카드/버튼/배지/텍스트 스타일이 일관된 구조

### L. 품질 기준

다음은 반드시 피해야 한다.

- 카드가 들쭉날쭉한 배치
- 화면마다 전혀 다른 앱처럼 보이는 스타일
- 과도한 색 사용
- 너무 싼 느낌의 운세 앱 디자인
- 한국어 문구가 어색하거나 길어서 깨지는 구성
- 전통 사주 느낌만 과도하게 강조되어 올드해 보이는 디자인
- 정보량이 많은 화면에서 정렬이 무너지는 구성

### M. 최종 목표

최종적으로 사용자는 이 앱을 보며 아래 인상을 받아야 한다.

- “정리 잘 된 프리미엄 사주 앱이다”
- “분석 메뉴가 한눈에 들어온다”
- “명식을 저장해두고 다시 보는 흐름이 편하다”
- “결과를 보고 추가 질문까지 이어지는 구조가 자연스럽다”
- “마이페이지와 기록 영역이 단순 설정창이 아니라 내 데이터 공간처럼 느껴진다”

이 기준으로 사주로 앱의 주요 모바일 화면들을 일관된 시스템으로 설계해줘.

## 4. 추가 화면 생성용 후행 지시문 템플릿

마스터 프롬프트 뒤에 아래 형식으로 붙여서 원하는 화면을 더 정확히 생성할 수 있다.

### 예시 1. 분석 허브 생성 요청
다음 화면을 우선 생성해줘.

- 화면명: 분석
- 모드: Light
- 목적: 카드형 분석 허브
- 2열 카드형 구조
- 상단 제목과 카테고리 필터 포함
- 카드 12개 이상 배치
- 무료/추천/인기 배지 반영
- 신한라이프 스타일처럼 정돈된 탐색형 느낌
- 사주로 브랜드 감성 유지

### 예시 2. 보관소 생성 요청
다음 화면을 생성해줘.

- 화면명: 보관소
- 모드: Dark
- 목적: 저장된 명식 관리
- 내 프로필 카드 + 즐겨찾기 + 저장된 명식 리스트 포함
- 각 카드에는 이름, 생년월일시, 관계 태그, 빠른 액션 버튼 포함
- 기존 사주로 감성을 유지하되 더 정돈되고 고급스럽게

### 예시 3. 명식 상세 생성 요청
다음 화면을 생성해줘.

- 화면명: 명식 상세
- 모드: Light
- 목적: 저장된 명식 정보를 보기 쉽게 요약
- 상단 본인 정보 카드
- 오행 카드 3개
- 시주/일주/월주/년주 격자
- 하단 고정 CTA: 운세 풀이 보기
- 복잡하지만 깔끔한 정보 설계

### 예시 4. 분석 결과 생성 요청
다음 화면을 생성해줘.

- 화면명: 금전운 분석 결과
- 모드: Dark
- 목적: 결과를 읽고 추가 질문하게 만드는 화면
- 요약 카드, 키워드 태그, 총평/강점/주의점/시기/조언 섹션 포함
- 하단에 추가 질문하기, 핵심만 요약, 쉽게 다시 설명 버튼 포함

## 5. 권장 생성 순서

피그마 Make에서는 한 번에 너무 많은 화면을 만들기보다 아래 순서로 나누는 것이 좋다.

### 1차
- 분석 허브
- 보관소
- 명식 상세

### 2차
- 홈
- 분석 결과
- 기록
- 마이페이지

### 3차
- 궁합 상대 선택
- 질문/답변 화면

- 더보기 액션 시트

## 6. 함께 넣으면 좋은 참고 지시문

필요하면 아래 내용을 추가로 덧붙인다.

### 디자인 시스템 일관성 강조
모든 화면은 같은 앱처럼 보이게 해줘.  
헤더, 카드, 버튼, 배지, 텍스트 위계를 통일하고, 화면별로 색상과 간격만 다르게 보정해줘.

### 사주 서비스 감성 강조
사주 앱이지만 너무 전통적이거나 촌스럽게 만들지 말고,  
현대적인 카드 UX 안에 한국형 운세 서비스 감성을 절제해서 녹여줘.

### 모바일 실사용성 강조
실제 모바일 앱에서 사용할 수 있게,  
텍스트 길이, 터치 영역, 카드 간격, 하단 CTA 위치가 현실적인 수준이 되게 해줘.

## 8. 결론

이 문서는 피그마 Make가 사주로 앱을 단순한 무료 운세 앱처럼 만들지 않고,  
**저장-분석-결과-질문-기록** 구조를 가진 프리미엄형 사주 앱으로 이해하도록 만드는 상위 통합 프롬프트이다.

가장 중요한 포인트는 다음과 같다.

- 분석 메뉴는 반드시 **카드형 허브**
- 보관소는 **명식 관리 허브**
- 결과 화면은 **추가 질문 유도 구조**
- 기록과 마이는 **개인 데이터 아카이브**
- 전체 화면은 **전면 라이트 모드 통일 전략**

이 문서를 기준으로 화면별 생성 프롬프트를 추가하면, 원하는 방향으로 훨씬 안정적인 결과를 얻을 수 있다.
