# 사주로(sajuro.ai) 앱 UI/UX 개편안  
## 피그마 Variables(컬러/타이포/간격/라운드) 토큰 표준안

---

## 1. 문서 목적

본 문서는 사주로(sajuro.ai) 앱의 UI/UX 개편을 위해 **피그마 Variables 기반 디자인 토큰 표준안**을 정의한 문서이다.  
첨부 화면들에서 보이는 다음 특성을 반영하여 설계하였다.

- **밝은 카드형 운세 메뉴 화면**: 여백이 넓고, 카드가 정렬된 구조
- **명식 상세 화면**: 정보 카드, 오행 컬러, 하단 CTA가 강조되는 구조
- **참고 서비스형 운세 메뉴 화면**: 섹션 제목과 카드형 탐색이 명확한 구조

본 표준안의 목적은 다음과 같다.

- 피그마 Variables를 활용하여 컬러/타이포/간격/라운드 기준을 통일
- 밝은 모드와 다크 모드를 함께 지원할 수 있는 토큰 체계 제시
- 사주로의 핵심 화면인 **분석 허브 / 명식 보관소 / 명식 상세 / 분석 결과 / 기록 / 마이페이지**에 재사용 가능한 디자인 기반 마련
- 추후 개발 handoff 시 디자인 시스템과 코드 토큰 간 매핑이 쉬운 구조 제공

## 2. 토큰 설계 원칙

### 2.1 핵심 원칙

1. **의미 기반 네이밍(Semantic naming)을 우선 적용**
   - 예: `color.bg.surface.primary`, `text.title.lg`
   - 단순 색상 이름보다 역할 중심으로 명명

2. **Primitive → Semantic → Component 3단계 구조**
   - Primitive: 기본 팔레트
   - Semantic: 역할 기반 토큰
   - Component: 특정 UI 컴포넌트 전용 토큰

3. **Light / Dark 모드 분리**
   - 기본 경험은 Light 기반 카드형 허브를 우선
   - 기존 사주로 감성인 Dark도 병행 지원

4. **사주로 특화 색상은 Accent/Meaning 레이어로 관리**
   - 오행, 무료/프리미엄 상태, 관계 배지 등은 별도 계층으로 분리

## 3. 피그마 Variables 권장 컬렉션 구조

피그마 Variables는 아래 6개 컬렉션으로 구성하는 것을 권장한다.

### 3.1 Collections

- `Primitives / Color`
- `Semantic / Color`
- `Primitives / Typography`
- `Semantic / Typography`

- `Primitives / Radius`

추가 확장용:
- `Primitives / Elevation`
- `Primitives / Motion`
- `Semantic / Status`
- `Semantic / FiveElements`

### 6.3 화면별 권장 간격 가이드

#### 카드형 분석 허브
- 화면 좌우: `24`
- 카드 간 수평/수직: `16`
- 섹션 제목과 카드 사이: `16`
- 섹션 간: `32`

#### 명식 상세 화면
- 상단 프로필 카드와 오행 카드 사이: `16`
- 오행 카드와 사주 격자 사이: `20`
- 하단 CTA 고정 바와 본문 사이: `24`

#### 마이/보관소 화면
- 프로필 카드 하단 간격: `24`
- 저장목록 카드 간: `12`
- 섹션 타이틀 간격: `20`

## 10. 피그마 모드 구성 권장안

### 10.1 Modes

컬러 Semantic Variables는 최소 2개 모드를 권장한다.

- `Mode / Light`
- `Mode / Dark`

### 10.2 Light/Dark 운영 원칙

- 라이트 모드: 전체 화면에 적용, 카드형 탐색 가독성 극대화
- 다크 모드: Variables에 정의는 유지하되, 향후 사용자 설정 또는 특수 화면에서 선택적 사용

### 10.3 추천 화면별 기본 모드

| 화면 | 권장 모드 |
|---|---|
| 홈 | Light |
| 분석 허브 | Light |
| 명식 보관소 | Light |
| 명식 상세 | Light |
| 분석 결과 | Light |
| 기록 | Light |
| 마이페이지 | Light |

> 모든 화면을 **전면 Light 모드로 통일**하여 앱 전체의 일관성과 가독성을 극대화한다. 프리미엄 감성은 골드/로즈 포인트 컬러와 절제된 타이포로 표현한다.

## 15. 결론

사주로의 디자인 토큰은 단순히 예쁜 화면을 만드는 기준이 아니라,  
앞으로 확장될 **분석 허브, 명식 보관소, 결과 리포트, 질문/답변, 마이페이지**를 하나의 체계로 묶는 운영 기준이다.

가장 중요한 방향은 다음과 같다.

- **전체 화면은 밝고 정돈된 카드 UX로 통일**
- **프리미엄 감성은 골드/로즈 포인트 컬러로 표현**
- **오행/상태/관계는 별도 의미 토큰으로 관리**
- **모든 화면은 Semantic Token 중심으로 연결**

이 기준으로 피그마 Variables를 먼저 정리해두면,  
추후 컴포넌트 확장과 개발 handoff가 훨씬 안정적으로 진행된다.
