# 사주로(sajuro.ai)  
## 핵심 화면 7종의 로우파이 와이어프레임 텍스트 명세

---

## 1. 문서 목적

본 문서는 사주로(sajuro.ai) 앱의 핵심 화면 7종에 대해,  
피그마 Make 또는 디자이너/기획자/개발자가 바로 참고할 수 있는 수준의 **로우파이 와이어프레임 텍스트 명세**를 정리한 문서이다.

이 문서의 목적은 다음과 같다.

- 화면별 정보 구조를 한눈에 이해할 수 있도록 정리
- 실제 피그마 설계 전 단계에서 배치와 우선순위를 텍스트로 고정
- 피그마 Make 결과물의 방향성을 더 정밀하게 통제
- 향후 상세 UI 설계, 컴포넌트 설계, 화면정의서 작성의 기반 마련

---

## 2. 작성 원칙

본 문서는 실제 픽셀 단위 설계서가 아니라,  
**모바일 앱 기준의 화면 구조와 영역 우선순위를 텍스트로 정의하는 로우파이 명세서**이다.

### 기본 원칙
- 각 화면은 **상단 → 본문 → 하단** 구조로 기술
- 각 화면은 **영역명 / 배치 / 주요 요소 / 문구 예시 / 인터랙션 포인트**로 정리
- 실제 디자인 시점에는 카드 높이, 여백, 타이포, 컬러는 디자인 시스템을 따른다
- 본 문서는 정보구조와 화면 논리를 우선한다

---

## 3. 대상 화면 7종

본 문서에서 다루는 핵심 화면은 아래 7종이다.

1. 홈
2. 분석 허브
3. 보관소
4. 명식 상세
5. 분석 결과
6. 기록
7. 마이페이지

# 4. 화면별 로우파이 와이어프레임 텍스트 명세

---

# 4.1 홈 화면

## 4.1.1 화면 목적
사용자가 앱에 진입하자마자 **오늘 볼 만한 분석과 최근 이어볼 콘텐츠**를 빠르게 확인하게 하는 진입 화면

## 4.1.2 화면 구조 개요
- 상단: 헤더 + 인사 문구
- 중단 상부: 오늘의 흐름 카드
- 중단 중부: 추천 분석 카드
- 중단 하부: 최근 본 분석 + 내 명식 바로가기
- 하단: 탭바

## 4.1.3 로우파이 배치 구조

```text
[상단 헤더]
  좌: 로고 또는 사주로
  우: 검색 / 알림

[인사 문구]
  "은우님, 오늘의 흐름을 살펴볼까요?"
  "지금 가장 궁금한 운세를 빠르게 확인해보세요."

[오늘의 한 줄 흐름 카드]
  제목: 오늘의 한 줄 흐름
  본문: 오늘은 판단보다 관찰이 유리한 날입니다.
  버튼: 오늘 운세 보기

[추천 분석 섹션]
  섹션 제목: 지금 많이 보는 분석
  2열 카드 4개
    - 내 사주 정밀분석
    - 운세 흐름
    - 금전운
    - 연인궁합
  하단 링크: 전체 분석 보기

[최근 본 분석 섹션]
  섹션 제목: 최근 본 분석
  리스트/카드 2~3개
    - 김은우 · 금전운
    - 쫑 · 연인궁합
    - 김은우 · 2026 신년운세

[내 명식 바로가기 섹션]
  섹션 제목: 내 명식 바로보기
  가로 카드 또는 소형 카드
    - 내 명식
    - 즐겨찾기
    - 새 명식 추가

[하단 탭바]
  홈 / 분석 / 보관소 / 기록 / 마이
```

## 4.1.4 주요 UI 요소
- 헤더
- 인사 텍스트
- 요약 카드 1개
- 분석 카드 4개
- 최근 본 기록 카드 2~3개
- 명식 바로가기 카드
- 탭바

## 4.1.5 인터랙션 포인트
- 오늘의 흐름 카드 탭 → 오늘 운세 또는 추천 분석 진입
- 추천 분석 카드 탭 → 분석 대상 명식 선택 또는 바로 분석 진입
- 최근 본 분석 탭 → 결과 이어보기
- 내 명식 카드 탭 → 보관소 또는 명식 상세 이동

# 4.2 분석 허브 화면

## 4.2.1 화면 목적
기존의 숨겨진 하단 메뉴 구조를 대체하는 **메인 분석 탐색 허브**

## 4.2.2 화면 구조 개요
- 상단: 헤더 + 필터 칩
- 본문 상부: 안내 문구
- 본문 메인: 2열 카드형 분석 메뉴
- 본문 하부: 추천 조합 또는 최근 본 분석
- 하단: 탭바

## 4.2.3 로우파이 배치 구조

```text
[상단 헤더]
  좌: 화면 제목 "분석 메뉴"
  우: 검색 / 필터

[안내 문구]
  "보고 싶은 분석을 선택해보세요."
  "사주·운세·궁합·생활 해석까지 한눈에 볼 수 있어요."

[카테고리 필터 칩]
  전체 / 내 사주 / 테마운 / 궁합·관계 / 생활·부가 / 이미지 분석

[분석 카드 그리드]
  2열 카드형 메뉴
  1행:
    - 만세력
    - 내 사주 정밀분석
  2행:
    - 평생운세
    - 운세 흐름
  3행:
    - 신년운세
    - 토정비결
  4행:
    - 금전운
    - 애정운
  5행:
    - 직업운
    - 사업운
  6행:
    - 건강운
    - 친구운
  7행:
    - 연인궁합
    - 결혼궁합
  8행:
    - 꿈해몽
    - 이름풀이
  9행:
    - 손금 보기
    - 관상 보기

[추천 조합 섹션]
  "이 순서로 보면 더 좋아요"
  - 정밀분석 → 운세 흐름 → 금전운
  - 내 사주 → 애정운 → 연인궁합

[하단 탭바]
  홈 / 분석 / 보관소 / 기록 / 마이
```

## 4.2.4 분석 카드 내부 구성
```text
[아이콘]
[제목]
[설명 1줄]
[우상단 배지]
```

예시:
- 제목: 금전운
- 설명: 재물 흐름과 지출 위험을 분석
- 배지: 현재 무료

## 4.2.5 인터랙션 포인트
- 카드 탭 → 명식 선택 바텀시트 호출
- 카테고리 칩 탭 → 해당 그룹만 필터링
- 추천 조합 탭 → 순차 진입 유도

# 4.3 보관소 화면

## 4.3.1 화면 목적
저장된 명식을 관리하고, 빠르게 분석 또는 궁합으로 이어지는 **명식 허브**

## 4.3.2 화면 구조 개요
- 상단: 헤더
- 본문 상부: 내 프로필 카드
- 본문 중부: 즐겨찾기
- 본문 하부: 저장된 명식 리스트
- 하단: 탭바

## 4.3.3 로우파이 배치 구조

```text
[상단 헤더]
  좌: 뒤로가기 또는 탭형 진입 시 없음
  중앙: 명식 보관소
  우: 검색 / 추가 / 정렬

[내 프로필 카드]
  이름: 김은우
  생년월일시: 1970/01/14 03:10 (음) · Seoul
  태그: 기본 명식
  빠른 버튼:
    - 정밀분석
    - 운세 흐름
    - 금전운

[즐겨찾기 섹션]
  섹션 제목: 즐겨찾기
  소형 카드/원형 아바타 형태 4~5개

[저장된 명식 섹션]
  섹션 제목: 저장된 명식
  우측 텍스트 버튼: 새 명식 추가

[명식 카드 리스트]
  카드 1:
    이름: 쫑
    정보: 1973/02/09 11:00 (양) · Seoul
    관계 배지: 연인
    최근 분석: 연인궁합
    버튼: 분석 / 궁합 / 더보기

  카드 2:
    이름: 누나
    정보: 1966/04/03 18:00 (음) · Seoul
    관계 배지: 가족
    최근 분석: 가족 궁합
    버튼: 분석 / 궁합 / 더보기

  카드 3:
    이름: 친구A
    정보: 1978/07/21 08:30 (양) · Seoul
    관계 배지: 지인
    최근 분석: 없음
    버튼: 분석 / 궁합 / 더보기

[하단 탭바]
  홈 / 분석 / 보관소 / 기록 / 마이
```

## 4.3.4 명식 카드 내부 구조
```text
[좌측 아바타]
[이름]
[생년월일시/지역]
[관계 배지]
[최근 분석]
[우측 또는 하단 액션 버튼 3개]
```

## 4.3.5 인터랙션 포인트
- 카드 본문 탭 → 명식 상세
- 분석 버튼 → 분석 허브 또는 직전 분석 진입
- 궁합 버튼 → 궁합 상대 선택
- 더보기 버튼 → 액션 시트 호출

# 4.4 명식 상세 화면

## 4.4.1 화면 목적
저장된 명식의 핵심 정보를 **보기 쉬운 요약형 구조**로 보여주고, 운세 풀이로 이어지게 하는 화면

## 4.4.2 화면 구조 개요
- 상단: 헤더
- 본문 상부: 인물 정보 카드
- 본문 중부: 오행/일주동물/성향 카드
- 본문 하부: 사주 격자
- 하단: 고정 CTA

## 4.4.3 로우파이 배치 구조

```text
[상단 헤더]
  좌: 뒤로가기
  중앙: 신년운세 또는 내 사주
  우: 안내 / 공유

[인물 정보 카드]
  이름: 김은우 (본인)
  생년월일: 1970.02.15 (양력)
  출생시간: 03시 00분
  성별: 남성
  우측 버튼: 변경

[요약 카드 3개]
  카드 1:
    상단 원형 컬러 아이콘
    제목: 화
    보조: 오행

  카드 2:
    이미지/아이콘
    제목: 붉은 호랑이
    보조: 일주 동물

  카드 3:
    상단 원형 라벨
    제목: 식신
    보조: 타고난 성향

[사주 격자 카드]
  상단 라벨 행:
    시주 / 일주 / 월주 / 년주

  각 열 구성:
    [십성]
    [천간 타일]
    [지지 타일]
    [하단 메타 정보]

  예시:
    시주:
      상관
      己
      丑
      상관 / 양

    일주:
      일원
      丙
      寅
      편인 / 장생

[하단 고정 CTA]
  버튼: 운세 풀이 보기
```

## 4.4.4 오행/격자 색상 규칙
- 목: 블루
- 화: 레드
- 토: 옐로우
- 금: 라이트 그레이
- 수: 블랙/다크

## 4.4.5 인터랙션 포인트
- 변경 버튼 → 명식 수정
- CTA 버튼 → 분석 결과 또는 분석 메뉴 진입
- 일부 오행/성향 카드 탭 → 상세 설명 툴팁 가능

# 4.5 분석 결과 화면

## 4.5.1 화면 목적
분석 내용을 읽고, 저장하고, 질문으로 이어지게 하는 **결과 중심 화면**

## 4.5.2 화면 구조 개요
- 상단: 헤더
- 본문 상부: 결과 요약 카드
- 본문 중부: 키워드 태그 + 섹션형 결과 본문
- 본문 하부: 관련 분석 추천
- 하단: 고정 액션 버튼

## 4.5.3 로우파이 배치 구조

```text
[상단 헤더]
  좌: 뒤로가기
  중앙: 금전운 분석 결과
  우: 저장 / 공유

[결과 요약 카드]
  제목: 김은우님의 금전운 분석
  핵심문장:
    "올해는 수입 확대보다 지출 관리와 구조 정비가 중요한 흐름입니다."
  키워드:
    - 재물 흐름
    - 지출 관리
    - 기회 포착

[키워드 태그 영역]
  태그 3~5개

[결과 본문 섹션]
  섹션 1: 총평
    "현재는 무리한 확장보다 흐름 정비가 더 중요합니다."

  섹션 2: 강점
    - 정보 포착 감각이 살아 있음
    - 재물 기회를 보는 감이 좋음

  섹션 3: 주의점
    - 즉흥 지출 위험
    - 과신에 의한 손실 주의

  섹션 4: 시기 포인트
    - 상반기: 정리
    - 하반기: 기회 확대

  섹션 5: 실행 조언
    - 큰 결정 전 구조를 먼저 점검
    - 소액 흐름부터 안정화

[관련 분석 추천 섹션]
  카드 2개 내외
    - 운세 흐름도 함께 보기
    - 사업운 이어서 보기

[하단 고정 액션 영역]
  버튼 1: 추가 질문하기
  버튼 2: 핵심만 요약
  버튼 3: 쉽게 다시 설명
  버튼 4: 관련 분석 추천
```

## 4.5.4 인터랙션 포인트
- 추가 질문하기 → 질문/답변 화면 진입
- 핵심만 요약 → 요약 버전 표시
- 쉽게 다시 설명 → 난이도 낮춘 버전 표시
- 저장 → 기록에 보관

# 4.6 기록 화면

## 4.6.1 화면 목적
저장된 분석 결과와 궁합 결과, 질문/답변 이력을 다시 볼 수 있는 **개인 분석 아카이브**

## 4.6.2 화면 구조 개요
- 상단: 헤더 + 필터
- 본문: 기록 카드 리스트
- 하단: 탭바

## 4.6.3 로우파이 배치 구조

```text
[상단 헤더]
  좌: 화면 제목 "분석 기록"
  우: 검색 / 정렬 / 필터

[필터 칩]
  전체 / 내 분석 / 궁합 / 질문답변 / 저장한 결과

[기록 카드 리스트]
  카드 1:
    제목: 김은우 · 2026 신년운세
```

# 4.7 마이페이지 화면

## 4.7.1 화면 목적
명식, 분석 기록, 설정, 결제, 문의까지 모아보는 **개인 데이터 관리 허브**

## 4.7.2 화면 구조 개요
- 상단: 헤더
- 본문 상부: 프로필 요약 카드
- 본문 중부: 메뉴 그룹 1~4
- 하단: 탭바

## 4.7.3 로우파이 배치 구조

```text
[상단 헤더]
  좌: 화면 제목 "마이페이지"
  우: 설정

[프로필 요약 카드]
  이름: 김은우
  설명:
    "저장된 명식과 분석 기록을 한곳에서 관리하세요."
  버튼:
    - 프로필 수정
    - 알림 설정

[메뉴 그룹 1: 내 보관함]
  - 명식 보관소
  - 즐겨찾기
  - 최근 본 분석

[메뉴 그룹 2: 분석 관리]
  - 분석 기록
  - 저장 결과
  - 질문/답변 내역
  - 결제한 리포트

[메뉴 그룹 3: 서비스 설정]
  - 알림 설정
  - 이용권/구독
  - 결제 내역
  - 데이터 관리

[메뉴 그룹 4: 지원]
  - 문의하기
  - 자주 묻는 질문
  - 공지사항
  - 서비스 소개

[하단 탭바]
  홈 / 분석 / 보관소 / 기록 / 마이
```

## 4.7.4 메뉴 항목 내부 구조
```text
[좌측 아이콘]
[항목 제목]
[필요 시 보조 설명]
[우측 화살표]
```

## 4.7.5 인터랙션 포인트
- 명식 보관소 → 보관소 화면 이동
- 분석 기록 → 기록 화면 이동
- 질문/답변 내역 → 질의응답 리스트 이동
- 구독/결제 → 구독 관리
- 문의하기 → 고객지원

## 5. 핵심 화면 간 연결 플로우

아래는 7개 화면 사이의 기본 연결 관계이다.

```text
홈
 ├─ 추천 분석 카드 → 분석 허브 또는 분석 결과
 ├─ 최근 본 분석 → 분석 결과
 └─ 내 명식 바로가기 → 보관소

분석 허브
 └─ 카드 선택 → 명식 선택 바텀시트 → 분석 결과

보관소
 ├─ 명식 카드 탭 → 명식 상세
 ├─ 분석 버튼 → 분석 허브 또는 분석 결과
 └─ 궁합 버튼 → 궁합 상대 선택

명식 상세
 └─ 운세 풀이 보기 → 분석 허브 또는 분석 결과

분석 결과
 ├─ 추가 질문하기 → 질문/답변
 ├─ 저장 → 기록
 └─ 관련 분석 추천 → 분석 허브

기록
 ├─ 다시보기 → 분석 결과
 └─ 대화 보기 → 질문/답변

마이페이지
 ├─ 명식 보관소 → 보관소
 ├─ 분석 기록 → 기록
 └─ 질문/답변 내역 → 질문/답변 리스트
```

## 7. 피그마 Make용 보조 문장

이 문서를 피그마 Make와 함께 사용할 경우, 아래 문장을 함께 넣으면 도움이 된다.

### 공통 보조 문장
- 각 화면은 정보량이 달라도 같은 앱처럼 보이게 해줘.
- 카드 간 간격, 제목 위계, 배지 스타일, 버튼 구조를 일관되게 맞춰줘.
- 너무 싸구려 무료 운세 앱처럼 보이지 않게, 프리미엄스럽고 정돈된 모바일 UI로 설계해줘.
- 한국어 문구가 실제 앱에서 자연스럽게 보이도록 길이를 조절해줘.

## 8. 결론

본 문서는 사주로 앱의 핵심 화면 7종을 **로우파이 와이어프레임 텍스트 형태로 구조화한 명세서**이다.

가장 중요한 포인트는 아래와 같다.

- 홈은 **진입과 재방문 유도**
- 분석 허브는 **카드형 탐색 중심**
- 보관소는 **저장 명식 관리 허브**
- 명식 상세는 **정보 요약 + CTA**
- 결과는 **읽기 + 질문 유도**
- 기록은 **개인 분석 아카이브**
- 마이는 **내 데이터 관리 허브**
- 전체 화면은 **전면 라이트 모드로 통일**

이 문서를 기반으로 피그마 Make 생성, 상세 UI 설계, 화면정의서 작성을 이어가면  
사주로의 개선 방향을 훨씬 안정적으로 구체화할 수 있다.
