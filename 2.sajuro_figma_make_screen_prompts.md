# 사주로(sajuro.ai)  
## 피그마 Make용 화면별 생성 프롬프트 세트

---

## 1. 문서 목적

이 문서는 피그마 Make에서 **사주로(sajuro.ai) 앱의 핵심 화면들을 개별 생성**할 때 사용할 수 있는 **화면별 프롬프트 세트**이다.

이 문서의 목적은 다음과 같다.

- 통합 마스터 프롬프트만으로 부족한 **화면별 정밀도 보완**
- 각 화면의 목적, 구성, 비주얼 스타일, 카피 방향을 개별적으로 통제
- 피그마 Make 결과물이 **사주로 개선안에 가깝게 나오도록 유도**
- 모바일 앱 기준의 현실적인 배치와 정보 구조를 반영

---

## 2. 사용 방법

### 권장 사용 순서
1. 먼저 **통합 마스터 프롬프트 md**를 상위 지시문으로 넣는다.
2. 그 다음 이 문서에서 필요한 화면 프롬프트를 복사해 화면별로 생성한다.
3. 한 번에 전체 앱을 만들지 말고, 화면 단위로 나누어 생성한다.
4. 생성 후 컴포넌트와 토큰을 정리하면서 반복 보정한다.

### 권장 생성 우선순위
1. 분석 허브
2. 보관소
3. 명식 상세
4. 분석 결과
5. 홈
6. 기록
7. 마이페이지
8. 질문/답변
9. 궁합 상대 선택
10. 바텀시트

---

## 3. 공통 후행 지시문

각 화면 프롬프트 끝에 아래 문장을 붙이면 결과 품질이 더 안정적이다.

### 공통 지시문
- 모든 텍스트는 한국어로 작성해줘.
- 실제 모바일 앱처럼 보이게 디자인해줘.
- 카드/버튼/배지/텍스트 스타일은 같은 앱처럼 일관되게 유지해줘.
- 과한 전통문양이나 촌스러운 운세 앱 느낌은 피하고, 현대적이고 정돈된 프리미엄 사주 앱처럼 보여줘.
- 카드 내부 여백과 정렬을 매우 깔끔하게 맞춰줘.
- 텍스트 길이는 실제 앱에서 어색하지 않도록 짧고 명확하게 유지해줘.

- 화면 목적: 사용자가 앱에 들어오자마자 오늘 볼 만한 운세와 최근 본 분석을 빠르게 이어보게 하는 화면

반드시 포함할 요소:
- 상단 헤더
- 인사 문구: “은우님, 오늘의 흐름을 살펴볼까요?”

- 추천 분석 카드 4개
- 최근 본 분석 영역
- 저장된 명식 바로가기 영역
- 하단 탭바

추천 분석 카드 예시:

- 운세 흐름
- 금전운
- 연인궁합

오늘의 한 줄 흐름 카드 예시 문구:
- “오늘은 판단보다 관찰이 유리한 날입니다.”
- “작은 신호를 놓치지 마세요.”

디자인 조건:
- 배경은 아주 연한 회색
- 카드는 흰색 또는 매우 옅은 회색
- 카드 간 정렬이 매우 중요
- 정보가 많아도 복잡해 보이지 않게 구성
- 홈 화면답게 가장 친절하고 진입 장벽이 낮아야 함

- 원본 분석 결과 요약 카드
- 추천 질문 칩
- 질문/답변 대화 버블
- 하단 입력창
- 보조 액션 버튼

추천 질문 예시:
- 언제 돈이 가장 잘 풀리나요?
- 투자운은 어떤가요?
- 주의해야 할 사람 유형은?
- 올해 큰 계약운은 있나요?

보조 액션 버튼 예시:
- 이 답변 저장
- 리포트에 반영
- 쉽게 설명
- 실행 조언만 보기

디자인 조건:
- 채팅 UI이지만 일반 메신저처럼 가볍지 않게
- 상단 요약 카드가 있어야 맥락이 유지됨
- 사주 앱의 프리미엄 질의응답 공간처럼 보여야 함

# 4.9 궁합 상대 선택 화면 생성 프롬프트

다음 모바일 앱 화면을 생성해줘.

- 앱명: 사주로(sajuro.ai)
- 화면명: 궁합 상대 선택
- 화면 목적: 궁합 분석 전 두 사람의 명식을 선택하는 화면
- 모드: Light
- 스타일: 단순하고 명확한 선택 화면
- 느낌: 입력 피로도는 낮고, 관계 분석으로 자연스럽게 이어지는 구조

반드시 포함할 요소:
- 상단 헤더 제목: 궁합 상대 선택
- 설명 문구
- 나 선택 카드
- 상대 선택 카드
- 저장된 명식 리스트 또는 선택 영역
- 하단 버튼:
  - 연인궁합 보기
  - 결혼궁합 보기
  - 가족궁합 보기

디자인 조건:
- 선택 흐름이 직관적이어야 함
- 카드 구분이 명확해야 함

## 6. 피그마 Make 입력 팁

- 한 번에 너무 많은 조건을 넣으면 결과가 퍼질 수 있으니, 핵심 화면부터 개별 생성한다.
- 먼저 전체적인 레이아웃을 맞춘 뒤, 텍스트/배지/카드 수를 보정한다.
- 모든 화면을 라이트 모드로 통일하여 일관된 탐색형 느낌을 유지한다.
- 생성 후에는 컴포넌트와 Variables를 정리해서 일관된 시스템으로 재조합한다.

## 7. 결론

이 문서는 피그마 Make가 사주로 앱의 각 화면을 **보다 구체적이고 안정적으로 생성하도록 돕는 화면별 프롬프트 세트**이다.

핵심은 다음과 같다.

- 분석은 **카드형 허브**
- 보관소는 **명식 관리 허브**
- 명식 상세는 **정보 밀도 높은 요약 화면**
- 결과는 **추가 질문 유도 화면**
- 기록과 마이는 **개인 데이터 아카이브**
- 전체 화면은 **전면 라이트 모드로 통일된 현대적이고 정돈된 프리미엄 한국형 사주 앱**

이 문서를 통합 마스터 프롬프트와 함께 사용하면, 원하는 방향의 화면을 훨씬 안정적으로 생성할 수 있다.
